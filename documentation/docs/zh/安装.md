# 安装部署

PracLab 由两层独立组件构成，可按需部署：

- **L1（必需）**：`PracLab.dll` — CounterStrikeSharp C# 插件，提供全部命令与 UI。
- **L2（可选）**：`PracLabReplayEngine.dll/.so` — Metamod C++ 插件，提供帧级移动录制/回放。未部署时仅 `.record/.replay` 系列命令不可用，其他功能不受影响。

> 构建产物可从 Release 下载，或参考[开发者文档](开发者.md)自行编译。

## 1. 前置依赖

| 依赖                                                                     | 版本       | 说明                                |
| ---------------------------------------------------------------------- | -------- | --------------------------------- |
| CS2 服务器                                                                | 最新版      | 专用服务器                             |
| [CounterStrikeSharp](https://github.com/roflmuffin/CounterStrikeSharp) | 1.0.371+ | CS2 脚本框架，须先部署到服务器                 |
| [Metamod:Source](https://www.sourcemm.net/)                            | 最新稳定版    | Layer 2 依赖（仅在部署回放引擎时需要）           |

## 2. 部署 L1 到 CS2 服务器

假设 CS2 服务器根目录为 `<CS2>`（即包含 `csgo/` 的目录）：

### 2.1 复制插件文件

```
<CS2>/csgo/addons/counterstrikesharp/plugins/PracLab/
├── PracLab.dll
└── lang/
    ├── zh-CN.json
    └── en.json
```

### 2.2 复制配置文件

```
<CS2>/csgo/cfg/PracLab/
├── config.cfg                 # 插件配置
├── prac.cfg                   # 练习模式配置
└── dryrun.cfg                 # 竞技模式配置
```

仓库内 `cfg/PracLab/` 目录已提供默认配置，直接整体复制即可。

### 2.3 启动服务器

启动后在控制台应看到：

```
[PracLab] Load: executing...
[PracLab] HH:mm:ss Core default language: zh-CN, plugin enabled: True (hotReload: False)
[PracLab] HH:mm:ss Core registered 9 map change commands, route table has XX keys
```

玩家在聊天栏输入 `.prac` 即可开启练习模式。

## 3. 部署 L2 到 CS2 服务器

### 3.1 复制动态库

**Windows**：

```
<CS2>/csgo/addons/PracLabReplayEngine/bin/win64/PracLabReplayEngine.dll
```

**Linux**：

```
<CS2>/csgo/addons/PracLabReplayEngine/bin/linuxsteamrt64/PracLabReplayEngine.so
```

### 3.2 复制 Metamod VDF 与 gamedata

仓库 [`replay-engine/configs/`](../../replay-engine/configs/) 目录提供现成文件：

```
<CS2>/csgo/addons/metamod/PracLabReplayEngine.vdf         # Windows
<CS2>/csgo/addons/metamod/PracLabReplayEngine.linux.vdf   # Linux（重命名为 .vdf）
<CS2>/csgo/addons/PracLabReplayEngine/gamedata.json       # 签名扫描配置
```

### 3.3 验证

启动服务器，控制台应出现：

```
[PracLab] HH:mm:ss Replay engine detected: PracLabReplayEngine loaded
```

或在 PracLab 控制台执行 `.currentrecord`，若提示「回放引擎未加载」则说明 Layer 2 未正确部署。

## 4. 升级与回滚

| 操作    | 步骤                                                                 |
| ----- | ------------------------------------------------------------------ |
| 升级 L1 | 覆盖 `PracLab.dll` 与 `lang/*.json` 等文件                               |
| 升级 L2 | 停止服务器 → 覆盖 `PracLabReplayEngine.dll/.so` → 启动服务器（Metamod 插件不支持热重载） |
| 回滚    | 替换为旧版本文件即可，配置文件向后兼容                                                |
