# 聊天栏命令

PracLab 通过拦截全局聊天（`say`）与队伍聊天（`say_team`）来识别玩家指令。所有命令均支持以下两种前缀形式，大小写不敏感：

| 前缀  | 示例      |
| --- | ------- |
| `.` | `.prac` |
| `!` | `!prac` |

***

| 命令      | 别名 | 功能                    |
| ------- | -- | --------------------- |
| `.prac` | —  | 加载练习配置，开启 practice 模式 |

### 出生点方框

| 命令            | 别名 | 功能                          |
| ------------- | -- | --------------------------- |
| `.showspawns` | —  | 重新显示双方出生点方框（prac 模式开启时自动显示） |
| `.hidespawns` | —  | 隐藏出生点方框                     |

> **E 键传送**：对准方框按下使用键（默认 E）即可传送到该出生点，聊天栏提示「已传送到第 N 个出生点」。CT 和 T 方框均为绿色，方框中心显示序号（按地图配置旋转）。prac 模式下不限制队伍，玩家可瞄准任一方框传送。方框仅显示竞技模式出生点（与 `.spawn` 系列命令一致）。

### Dryrun 与回合控制

| 命令              | 别名     | 功能                                       |
| --------------- | ------ | ---------------------------------------- |
| `.dryrun`       | `.dry` | 从 prac 模式临时切换到竞技配置打一个回合，回合结束自动回到 prac 模式 |
| `.restartround` | `.rr`  | 重新开始当前回合（执行 `mp_restartgame`）            |

> **Dryrun 流程**：踢出所有 bot → 关闭 prac 特有 ConVar（sv\_cheats、traj、infinite\_ammo 等）→ 加载 `dryrun.cfg` 竞技配置 → 重启回合开始；回合结束（`EventRoundEnd`）自动恢复 prac 配置。用于在练习中快速打一个真竞技回合。

### 回放系统

| 命令                  | 别名            | 功能                                                          |
| ------------------- | ------------- | ----------------------------------------------------------- |
| `.record`           | —             | 进入待录制状态，**玩家移动或按 F 键（检视键）开始录制**，再次按 F 或 `.stoprecord` 停止并保存 |
| `.stoprecord`       | —             | 停止录制并保存（与按 F 键等效）                                           |
| `.replay [Id]`      | —             | 无参数：并行回放所有已停止的录制；指定 Id：仅回放该录制。自动创建同队 Bot 作为回放载体             |
| `.stopreplay`       | —             | 停止所有正在进行的回放并踢出对应 Bot                                        |
| `.clearrecord <Id>` | —             | 删除指定 Id 的录制（同时删除 JSON 文件与列表条目）                              |
| `.clearrecordall`   | —             | 清除所有录制                                                      |
| `.currentrecord`    | `.currentrec` | 在**服务器控制台**输出录制列表表格（ID / 名称 / 状态 / Bot）                     |

> **二层架构**：回放系统由两部分组成——
>
> - **Layer 1（CSSharp C#）**：命令分发、UI 提示、录制文件 JSON 读写（`cfg/PracLab/recordings/`）。
> - **Layer 2（C++ Metamod 插件** **`PracLabReplayEngine`）**：通过 Hook `ProcessMovement`、`CCSBot::Update`、`PlayerRunCommand` 等引擎函数实现帧级移动录制与回放。
>
> 跨语言 ABI 契约见 [`replay-engine/include/praclab_replay.h`](../../../replay-engine/include/praclab_replay.h)，修改字段顺序/类型须同步更新两层代码。
>
> **回放引擎未部署时**：所有 `.record/.replay/...` 命令会向玩家提示「回放引擎未加载，回放功能不可用」，但插件其他功能不受影响。
